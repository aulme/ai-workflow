# Draft a proposal

**In:** short description from the human. **Out:** coherent first-draft file in `specs/proposals/`.

Read [`README.md`](./README.md) for the proposal-based workflow rules before running this procedure.

## Steps

1. **Clarify.** Ask as many questions as genuinely needed — problem, scope, hard constraints, preferred tech, non-goals, consumers. Don't guess on anything material.
2. **Check overlap.** If an open proposal substantially covers this, suggest extending it instead.
3. **Draft** `specs/proposals/<slug>.md` in the format from the workflow README, `Status: Draft`. Open questions are expected — they seed sharpening.
4. **Stop.** Surface the draft; ask whether to sharpen.

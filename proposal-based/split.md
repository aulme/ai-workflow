# Split the proposal

**In:** a proposal that feels large. **Out:** stages within it, or separate proposals.

Read [`README.md`](./README.md) for the proposal-based workflow rules before running this procedure.

## Steps

1. **Find natural seams** — points where you could ship, test, and have standalone value.
2. **Propose stages.** For each: short name · what it delivers · how it's independently testable · dependencies on prior stages.
3. **Stages vs. separate proposals.** Shared design thread → stages (add a `Stages` section). Only temporally adjacent → spawn separate proposals.
4. **Update** the file(s).
5. **Stop.** Confirm the split with the human.
